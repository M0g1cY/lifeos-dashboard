---

## Phase 3 目标

在 Phase 2 的基础上添加基础交互功能，让 Dashboard 从静态展示变成可交互的应用。

**核心原则**：
- 不引入 Zustand（只用 React Hooks）
- 不引入 LocalStorage（只做内存状态）
- 不新增复杂功能（只做基础交互）
- 保持视觉一致（不改变视觉效果）

## 风险管理

### 已知风险
1. **状态管理复杂度**：不使用 Zustand，可能导致状态管理复杂
   - **缓解措施**：使用 React Context 或 Props Drilling
2. **弹窗动画性能**：弹窗动画可能影响性能
   - **缓解措施**：使用 CSS 动画，避免 JavaScript 动画
3. **表单验证复杂度**：表单验证可能比较复杂
   - **缓解措施**：使用简单的验证逻辑，不引入 React Hook Form

### 应急方案
- 如果某个功能超过预计时间 50%，立即向 MagicY 报告
- 如果遇到技术难题，先尝试 2 种不同方案，仍无法解决则升级
- 如果发现需求不明确，立即暂停并与 Product Agent 沟通
